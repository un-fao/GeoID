Production setup (Cloud Run + Elastic Cloud)
  1. In Elastic Cloud, create an API key with read on dynastore-logs-* and the kibana_user application role scoped to the space containing the dashboard.
  2. Store the key in Secret Manager: 
    gcloud secrets create kibana-upstream-api-key --replication-policy=automatic
echo -n "$KEY" | gcloud secrets versions add kibana-upstream-api-key --data-file=-
  3. Mount it on the geoid Cloud Run service as KIBANA_UPSTREAM_API_KEY and set KIBANA_UPSTREAM_URL to the Elastic Cloud Kibana endpoint.
  4. If Elastic Cloud is on a private VPC / PSC, attach a Serverless VPC Connector to the geoid Cloud Run service.
  5. Restart the service. All four status dots should go green on the next page load.
Troubleshooting
Failing check Likely cause Verify
Elasticsearch ES_HOST/ES_PORT unset or unreachable from the geoid container curl $ES_HOST:$ES_PORT/_cluster/health
Dashboards upstream KIBANA_UPSTREAM_URL unset, wrong host, or network egress blocked curl $KIBANA_UPSTREAM_URL/api/status from the geoid container
Dashboard objects Auto-import failed (see warning logs for dashboards_provisioner) — usually an auth issue Restart the service; re-check this page
API key authorized KIBANA_UPSTREAM_API_KEY missing, expired, or lacks the required role Inspect the key in Elastic Cloud; generate a fresh one with the roles above
