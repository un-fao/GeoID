Deep Dive

Storage Abstraction Layer

How catalog maps logical Codes to physical Schemas, enabling renaming without data movement.

Database Topology

Understanding how Dynastore maps logical concepts to physical storage structures.

1. The Catalog Schema

Every Catalog has a **Physical Schema** resolved via its code. The logical code (e.g., `agriculture`) maps to a generated schema (e.g., `s_a1b2c3`). This ensures zero data leakage and allows `code` renaming.

CREATE SCHEMA IF NOT EXISTS "s_a1b2c3";

2. The Collection Table

A Collection becomes a Partitioned Table inside that schema. The logical code (e.g., `fields_v1`) maps to a generated table name (e.g., `t_x9y8z7`). Configuration for this table is stored in the same physical schema.

CREATE TABLE "s_a1b2c3"."t_x9y8z7" (...) PARTITION BY LIST (partition_key);

3. "Lazy" Partitions

The platform creates partitions Just-In-Time (JIT) using Database Triggers. When a row is inserted, the DB checks if a partition exists. If not, it creates it instantly.

TRIGGER BEFORE INSERT -> create_partition_if_not_exists()

Strategic Value

The Platform Value

The Agro-Informatics Platform (AIP) is a strategic asset designed to lower TCO, mitigate risk, and enable high-scale geospatial analysis across the enterprise.

Total Cost of Ownership (TCO)

Traditional GIS systems are monolithic license-heavy beasts. Our modularity means you deploy only what you need. By leveraging PostgreSQL capabilities (Partitioning, JSONB) instead of expensive external orchestrators, infrastructure costs are slashed by up to 60%.

Risk Mitigation

Monoliths are brittle; one bug brings down the system. Our "Three Pillars" isolation ensures that a failure in data processing never impacts customer-facing APIs. This structural resilience protects SLAs and brand reputation.

Ecosystem Agility

The geospatial landscape moves fast. New standards (like OGC API) emerge yearly. Our API-agnostic core allows adoption of new standards via plug-and-play Extensions without refactoring the engine. We are future-proof by design.

The Trillion-Row Scale

With the physical vs. logical abstraction layer, the platform handles data at a scale where competitors crumble. "Lazy" partitioning and base62-encoded physical schemas allow for infinite horizontal scaling of tenant data.
