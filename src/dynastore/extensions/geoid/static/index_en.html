Three Pillars Architecture

Strict separation of concerns prevents the "big ball of mud".

Modules

Backend-agnostic libraries. Single source of truth per data domain. One module owns one table.

Extensions

Stateless HTTP adapters. Translate requests into module calls. Plug in new OGC standards without touching the core.

Tasks

Isolated background workers exposed through OGC API – Processes. Ingestion, indexing, and analysis run in their own containers.

Standards Coverage

Live data — each badge fetches its OGC API conformance endpoint.

Capabilities

Multi-tenancy

Each catalog maps to a distinct PostgreSQL schema. Zero data leakage, rename without data movement.

OGC Native

Features (Parts 1-4), STAC 1.0.0, Coverages, Tiles, Maps, Processes, Records, DGGS, Connected Systems, Moving Features, Styles.

Trillion-row scale

Lazy JIT partitioning via DB triggers. Base62-encoded physical schemas. No upfront capacity planning.

ABAC / RBAC

Fine-grained, declarative authorization. Identity matchers + content-hash gating for write paths.

Asset registry

Stable URIs for every asset. Content-hash deduplication. Bridges STAC items to physical files.

OLAP Dimensions

Paginated datacube dimensions (research). Bridges OGC API to OLAP query patterns.
