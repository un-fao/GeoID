Catálogo geoespacial nativo OGC
a escala planetaria

Una plataforma de servicios de catálogo multitenant y compatible con OGC. Gestione billones de entidades en tenants aislados y expóngalas a través de todos los estándares OGC API principales.

Arquitectura de los Tres Pilares

La separación estricta de responsabilidades evita el "big ball of mud".

Módulos

Bibliotecas independientes del backend. Fuente única de verdad por dominio. Un módulo es dueño de una tabla.

Extensiones

Adaptadores HTTP sin estado. Traducen solicitudes en llamadas a módulos. Conecte nuevos estándares OGC sin tocar el núcleo.

Tareas

Workers en segundo plano aislados expuestos a través de OGC API – Processes. La ingesta, la indexación y el análisis corren en sus propios contenedores.

Cobertura de estándares

Datos en vivo — cada insignia consulta su endpoint OGC API conformance.

Capacidades

Multitenancy

Cada catálogo se asigna a un esquema PostgreSQL distinto. Cero filtración de datos, renombrado sin movimiento de datos.

OGC nativo

Features (Partes 1-4), STAC 1.0.0, Coverages, Tiles, Maps, Processes, Records, DGGS, Connected Systems, Moving Features, Styles.

Escala de billones de filas

Particionado JIT mediante triggers. Esquemas físicos en base62. Sin planificación de capacidad anticipada.

ABAC / RBAC

Autorización declarativa de grano fino. Identity matchers + content-hash gating para las rutas de escritura.

Registro de assets

URIs estables para cada asset. Deduplicación por content-hash. Une los items STAC con los archivos físicos.

Dimensiones OLAP

Dimensiones de datacube paginadas (research). Conecta OGC API con patrones de consulta OLAP.
